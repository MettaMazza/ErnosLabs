Generative UI · zero JavaScript

The AI Canvas

Type what you want and a local model builds the entire interface — as raw HTML, CSS and SVG, streamed straight to your browser. No framework, no client-side JavaScript: every button, link and game move is a plain HTML <form> that posts back, and the engine answers with the next screen. It's an ErnosDecent feature, written in ErnosPlain, running live on the source machine.

Checking the canvas…

The model is rendering your interface…

🌙

The canvas is served live from Maria's machine. It's asleep right now — come back when the host is online and it'll build whatever you ask for.

Try: “a Connect 4 game”, “a calculator”, “a neon synthwave sign-in card”, “a wikipedia-style page about the Iron Age with clickable links”. It runs on a capable local model (gpt-oss-20b) dedicated to the canvas — you'll see it inferring, then the screen appears. The frame is script-sandboxed: nothing the model emits can run code.

How it works

The framework is the model

Instead of shipping a heavy front-end to parse Markdown and manage state, the engine emits finished HTML/CSS/SVG and streams it into the page. State lives in the forms themselves — every screen encodes what to do next — so there's no client bloat and no server-side session to keep. It's the local-first idea taken to its limit: native browser capabilities plus direct model output, nothing in between.
